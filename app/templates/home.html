End-to-end sequence

How the application reasons

1

Capture context

Weather autofill reduces manual entry by using browser permission and a live weather API call.

2

Predict crop prior

The CSV model scores probable crops from environmental and soil features.

3

Analyze the leaf

The vision model identifies healthy vs disease classes from PlantVillage-style labels and field imagery.

4

Deliver explanation

The final page shows confidence, disease treatment, crop support hints, and a Grad-CAM highlight map.

Deployment checklist

Recommended production setup

  • Set `OPENWEATHER_API_KEY` before enabling live weather autofill.
  • Install TensorFlow to activate the disease and hybrid inference routes.
  • Use HTTPS in production because browser geolocation requires a secure context.
  • Keep disease artifacts under `models/disease/` and generated overlays under `app/static/generated/`.
